Step-by-Step Instructions: Shower Study

Data Button - Shower Button - Searching Data Files - Selecting Data - Analyzing
Shower Candidates - View Plot Controls - Change Caption - The Plot - Save Plot - Done

  • Keep two windows open, one with these instructions and the other with the shower study web pages.
  • Go back and forth between them.
  • Use the scroll bar to move the instructions.
  • Look for the blue highlights for guidance.

We shall create a plot with data from the Fermilab Test Array dated 09/01/2016 to 09/02/2016 (DD/MM/YYYY). We entered Fermilab Test Array into the e-Lab as a school even though it does not seem like a typical school.

Step 1:Click on the Data button in the main menu.
Step 2:Click on the Shower button in the submenu.
Step 3.1:Look at the search page. On the right, there are links to a tutorial, step-by-step instructions, FAQs and related milestones. The legend shows icons for information about each data file.
Step 3.2:We want to find all data for the Fermilab Test Array for Sep. 1 and 2, 2016. We had two detectors in the same room collecting data simultaneously.
a) Select School in the pulldown list
b) Enter Fermilab Test Array in the search field.
c) Click on the little arrow next to Advanced Search.
Step 3.3:a) Select Start Date in the pulldown list for dates and enter '09/01/2016' in the first date field and '09/02/2016' in the second date field. One can also use the calendar.
b) Choose Blessed: All. This choice will get all the data files, blessed or not. Note: Default implies blessed files only (the ones that passed the "blessed" categorization process).
c) Click Search Data.
Step 3.4:a) Click on the little arrow next to 'Fermilab Test Array' to see the data files.
Step 3.5:a) Click on the little arrow next to 'September 2016' to see the data files for that month.
Step 3.6:Notice there are data for two detectors and that they are both in an array configuration. See the data by clicking on the magnifying glass icon next to the date, and see additional information by clicking on the comments bubble.
Step 4:a) Click all the checkboxes associated with detector 6119 and 6148. One can also use the 'All' option.
Step 4.1: a) Click Run shower study on the right.
Step 4.2:Look at the display of all the selected data files. One can view the raw data, check the statistics, and compare them by using the links on the right. Also check that the geometry files exist; one cannot do a shower study without them.
Step 5:To understand the input parameters, click on the question mark next to each one.
a) Set the 'Center of graph view' to the detector id, 6119.
b) Set 'Event Gate' to 1000 ns.
c) Set 'Detector Coincidence' to 2.
d) Set 'Channel Coincidence' to 2
e) Set 'Hit Coincidence' to 2.
f) Click Analyze. This may take some time, so don't be alarmed if the plot does not appear right away.
Step 5.1:If your analysis takes very little time to run, the plot may show up quickly. More often, though, one gets a progress bar displaying how the analysis is doing. One can click on Queue study to allow the analysis to continue in the background. One can click on Analysis output to see the inner workings. This might help if there is an error in the analysis.
Step 5.2:Click on Analyses on the navigation bar or analysis list in the text to see the job's progress.
Step 5.3:When the job is done, one will see an entry in the Analysis List like this. Click on the link to see the plot.
For the shower study there are actually two links - one for the Shower Study, and the other for each Event Plot.
Step 6:Here are a list of the possible shower candidates. The plot is on the right. The user can click on the event date and display the plot associated with each event. The data table shows all the events associated with each candidate.
a) Choose the candidate with Event Date Sep 1, 2016 01:02:17.
b) Click on the plot on the right to see a bigger version. Notice that the caption (all those words at the top) is very messy.
c) Click Change at the bottom to go back and fix the caption in the plot parameters.
Step 7:a) Click on the small arrow to see the Plot Controls, and change the caption.
Step 8: a) Change the z-max to 70. This will eliminate extra (later) hits from 6119 counters.
b) Edit the list of data files from the Figure caption, and replace it with something simpler such as:
Data: Fermilab Test Array
Sep 01-02, 2016
Detectors: 6119, 6148
c) Keep the other settings, and click Analyze.
New caption
Step 9:a) Click on the small plot to open a window with this plot.
b) Scroll down in the plot window past the date to save the plot permanently.
Step 10:Enter a name for the plot similar to, but not the same as, the one shown. Then, click Save Plot. Each time through this tutorial, the user should choose a new name for the plot. It will be available for your poster.
Step 11:Note the successfully saved plot. Click Close. Also close the window with the plot.
The user can create more plots for other shower candidates or go back to choosing a new data set by clicking Shower in the menu. At any time, one can click View Plots to see the saved plots.
Step 12: This step onward is optional.
a) Click on the box by 'View Multiplicity Totals.'
b) A new column called 'Multiplicity Totals' appears listing the total number of counters hit in the event. This can be less than the number of hits.
c) With the pulldown menu under 'Multiplicity Totals', the user can select only events with a specific number of counters hit (multiplicity). The default is 'All'.
d) Notice there are 1376 candidates.
Step 13: Multiplicity 6.
a) Select '6' in the pulldown menu.
b) Now, only 171 candidates with exactly 6 hit counters are listed.
Notice that events come in three varieties in hit counters in DAQs 6119 and 6148:
c) (4,2)
d) (3,3)
e) (2.4)
  • One can make a plot by hand of frequency (y variable) vs. the multiplicity (x variable).
  • This can be related to the number of muons but depends on geometry (2 stacked counters count 1 muon, whereas 2 horizontally separated counters count 2 muons).
